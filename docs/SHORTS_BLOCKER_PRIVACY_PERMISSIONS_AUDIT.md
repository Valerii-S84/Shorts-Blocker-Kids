# Shorts Blocker Kids Privacy And Permissions Audit

Status: Partial

## Scope

- Inspected debug and release merged manifests generated by local Gradle tasks.
- Confirmed requested permissions and privacy-sensitive capabilities for the
  current local debug and release build outputs.
- Built the release Android App Bundle locally.

## Permissions Requested

From the merged debug manifest:

- `com.android.vending.BILLING`
- `com.shortsblockerkids.debug.DYNAMIC_RECEIVER_NOT_EXPORTED_PERMISSION`
- `android.permission.ACCESS_NETWORK_STATE`
- `android.permission.INTERNET`

From the merged release manifest:

- `com.android.vending.BILLING`
- `com.shortsblockerkids.DYNAMIC_RECEIVER_NOT_EXPORTED_PERMISSION`
- `android.permission.ACCESS_NETWORK_STATE`
- `android.permission.INTERNET`

Notes:

- `android.permission.BIND_ACCESSIBILITY_SERVICE` is declared as the service binding permission on `ShortsBlockerAccessibilityService`; it is not requested as a `<uses-permission>`.
- `android.permission.DUMP` appears as a permission protecting an AndroidX ProfileInstaller receiver; it is not requested as a `<uses-permission>`.

## Privacy Confirmation

- `android.permission.INTERNET` and
  `android.permission.ACCESS_NETWORK_STATE` are present through the Google Play
  Billing SDK only.
- No `CAMERA` permission.
- No `RECORD_AUDIO` / microphone permission.
- No location permission.
- No contacts permission.
- No SMS permission.
- No watch history storage.
- No account system.
- Billing backend is limited to Google Play subscription verification.
- No cloud sync.
- No analytics.
- No ads.
- Google Play Billing is integrated for subscriptions.
- The app stores local subscription entitlement status and last verification
  timestamp. The billing backend stores only billing technical entitlement data.
- No language/content analysis.
- No video/audio analysis.
- No raw accessibility tree logging in release code paths.

## Accessibility Data Handling

- Detector logging is guarded by `BuildConfig.DEBUG`.
- Debug dashboard detector status is guarded by `BuildConfig.DEBUG`.
- The scanner keeps only sanitized signals: class names, view IDs, generic known content-description signals, and geometry.
- Non-YouTube events are used only to dismiss/reset overlay state or consume a debug-only overlay request.
- The service reads `rootInActiveWindow` only for the YouTube package.

## Known Limitations

- This audit covers local debug and release build outputs, not a Play Console
  accepted artifact.
- AndroidX may add implementation permissions or protected components in merged manifests.
- Final Play review still requires detailed device matrix validation and policy
  review of Accessibility usage.
- Final Play review still requires hosted Privacy Policy URL, Play Console Data
  Safety submission, Accessibility declaration submission, and demo video URL.
