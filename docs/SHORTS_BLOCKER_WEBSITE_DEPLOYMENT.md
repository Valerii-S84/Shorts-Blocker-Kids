# Shorts Blocker Kids Website Deployment

Status: Partial. Static website source is implemented locally. Live deployment
to `https://shortsblockerkids.de` is blocked until owner/server/DNS setup is
available and verified.

## Scope

Public website:

```text
https://shortsblockerkids.de
https://shortsblockerkids.de/privacy
https://shortsblockerkids.de/support
https://shortsblockerkids.de/terms
```

Billing backend remains separate:

```text
https://billing.shortsblockerkids.de
```

The public website must not expose backend admin, debug, credentials, Play
Developer API details, or server internals.

## Source

```text
website/
```

uses the existing Android app icon as favicon/app icon support.

## Deployment

Deploy the contents of `website/dist/` to the HTTPS virtual host for
`shortsblockerkids.de`.

Minimal Nginx shape:

```nginx
server {
    listen 443 ssl http2;
    server_name shortsblockerkids.de www.shortsblockerkids.de;

    root /var/www/shortsblockerkids.de/dist;
    index index.html;

    location / {
        try_files $uri $uri/ =404;
    }
}
```

Redirect `http://` to `https://` and either redirect `www` to the canonical
host or keep both hosts serving the same static files with canonical metadata
pointing to `https://shortsblockerkids.de`.

## Google Play Readiness Checks

Before Play Console submission, verify in a normal browser:

```text
https://shortsblockerkids.de
https://shortsblockerkids.de/privacy
https://shortsblockerkids.de/support
```

Required live checks:

- HTTPS certificate is valid.
- Pages load without login, geofence, PDF-only access, or crawler blocking.
- Privacy page matches current app behavior and Data Safety docs.
- Support page shows `svalerii535@gmail.com`.
- No fake user counts.
- No fake testimonials.
- No direct APK download as the primary install action.
- CTA remains `Coming soon on Google Play` until a live Play or testing URL is
  configured.

## Metrics Widget

The home page metrics widget intentionally has no live endpoint configured.
It shows neutral private-testing copy.

If a metric endpoint is added later, it must be separate from billing admin
surfaces and return verified real data only:

```json
{
  "verified": true,
  "privateTestingFamilies": 0,
  "blockedEvents": 0,
  "generatedAt": "2026-05-31T00:00:00Z"
}
```

Do not publish placeholder install counts, fake active-user numbers, fake
blocked-event counts, or generated testimonials.
